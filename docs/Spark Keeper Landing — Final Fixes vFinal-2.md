Spark Keeper Landing — Final Fixes vFinal-2

1) Cursor swarm: scroll-behavior “скаче” (прибрати джиттер)

Problem

Mouse-follow ок, але при scroll swarm “стрибає/скаче” негарно — проблема лишилась.

Requirement

Зробити плавну інерцію на scroll без джиттера:
	•	swarm має мати дві компоненти руху:
	1.	cursor-follow (позиція курсора)
	2.	scroll inertia (відставання/доганяння при зміні scrollY)
	•	scroll не повинен “телепортувати” рій разом з курсором

Implementation guideline (щоб деву було ясно)
	•	Не прив’язувати swarm позицію напряму до clientY під час scroll.
	•	Замість цього:
	•	тримати targetPos = cursorPos у world space
	•	тримати currentPos і оновлювати через smoothing (lerp/spring)
	•	на wheel/scroll додавати scrollVelocity як тимчасовий offset, який затухає (decay)
	•	Обов’язково: один requestAnimationFrame loop, без “setState на кожен wheel event”.

Acceptance criteria
	•	При швидкому scroll (trackpad) рій не смикається і не “стрибає”.
	•	Видно легке відставання і доганяння (інерція), але без різких ривків.
	•	При зупинці scroll рій плавно стабілізується біля курсора.
	•	Нема “липкої плями”, яка перескакує разом зі сторінкою.

2) Thought phrases (“This was important / Lost in screenshots”): randomization + min duration

Current state

Стало краще, але:
	•	інколи фрази зникають занадто швидко
	•	“рандомізацію живості” повністю не зроблено (передбачуваний патерн)

Requirements
	1.	Встановити мінімальні пороги, щоб не було “миттєвих” появ:

	•	minVisibleTime (мінімальний час, коли фраза реально читається)
	•	minFadeIn, minFadeOut (мінімальні тривалості fade)

	2.	Рандомізація для кожного циклу кожної фрази:

	•	позиція (в safe-area, щоб не лізло під заголовки)
	•	delay
	•	duration
	•	drift vector (x/y)
	•	opacity curve (але з вимогою: може падати до 0, але не миттєво)

	3.	Асинхронність:

	•	фрази не повинні оновлюватись “пачкою”

Acceptance criteria
	•	Нема випадків, коли фраза “зʼявилась і зникла майже одразу”.
	•	Є мінімальний час читабельності для кожної появи.
	•	Одна і та сама фраза зʼявляється в різних місцях з різними таймінгами.
	•	Патерн виглядає як “живі думки”, не як повторюваний цикл.

3) “Universe core” (помаранчева діра): замінити складні лінії на “solar flares”

Change request (оновлення вимог)

Відмовляємось від ліній, які “прив’язані до конкретних слів” (бо складно і дорого).

Замість цього:
	•	навколо помаранчевого ядра зробити анімовані промені/спалахи, як у Сонця:
	•	різна довжина (рандом/періодика)
	•	різна товщина/яскравість
	•	пульсація (scale/opacity)
	•	легке “тремтіння” (micro noise)
	•	ці промені мають створювати ілюзію напрямку “всмоктування” без прив’язки до слів

Acceptance criteria
	•	Ядро — помаранчеве (як раніше), читається як “центр”.
	•	Є flare-промені навколо ядра (мінімум 8–16 штук або procedural).
	•	Промені пульсують (не статичні), довжина/яскравість варіюється.
	•	Виглядає як “енергія/спалахи”, а не як статичні рисочки.
	•	Не потрібно синхронізувати промені зі словами (але вони мають підсилювати відчуття руху в центр).
