Spark Keeper Landing — Tech Spec vNext

Scope: polish + motion depth + testimonials + cursor swarm

Ціль

Довести лендінг до “shipping quality” без втрати вашого sci-fi/void стилю: більше живих думок/іскор, коректні анімації “збирання в ядро”, не-card testimonials, і новий cursor-ефект.

⸻

1) Cursor interaction: “Spark Swarm Tail”

Change request

Замість “весь бекграунд тягнеться за курсором” — зробити локальне “роїння” біля курсора:
	•	біля мишки зʼявляється “хвіст” із spark-частинок
	•	хвіст тягнеться із затримкою, з bounce/elastic
	•	форма хвоста “поліморфна” (не пряма лінія), як рій/вихор

Acceptance criteria
	•	Ефект помітний одразу на desktop при русі мишкою.
	•	Рій локальний (радіус впливу ~120–220px), не рухає весь бекграунд.
	•	Є “tail”: частинки відстають на 80–180ms і згасають на відстані.
	•	Має “bounce”/elastic відчуття (але без смикань).
	•	Performance: не падає FPS; на слабких пристроях ефект деградує (менше частинок/нижча частота).
	•	На mobile: вимкнено або замінено на subtle idle (без “мишки”).

⸻

2) Thought cloud block: “Save it somewhere…”

Change request

Зараз думок мало (~6), вони надто тихі, opacity змінюється повільно. Треба:
	•	збільшити кількість думок (орієнтир 12–18)
	•	зробити більшу амплітуду руху (вгору/вниз/вліво/вправо)
	•	зробити ефект “спалах → згас до 0”, а не повільний fade
	•	кожна думка анімується незалежно (рандомні delay/duration/траєкторії)

Acceptance criteria
	•	На екрані одночасно видно ≥ 10 думок (не 6).
	•	Кожна думка має власний таймінг (немає синхронної пульсації).
	•	Є цикл: opacity піднімається до читабельного рівня → падає до 0 (повністю зникає).
	•	Рух помітний (амплітуда не “ледь-ледь”), але не заважає читанню основних заголовків.
	•	Нема “порожньої висоти”: заповнення блоку виглядає природно, не так що елементи сконцентровані зверху, а знизу порожньо.

3) Core “magnet” block: слова збігаються в пульсуюче ядро

Change request
	1.	Замінити тексти: тут мають бути “реальні” думки/наміри (а не дубль із попереднього блоку).
Приклади (EN, можна міксувати):
	•	“Book: The Courage to Be Disliked”
	•	“Plan: Iceland in winter”
	•	“Gift idea: classical concert tickets”
	•	“Watch: True Detective S1”
	•	“Read later: link about RWA tokenization”
	•	“Buy: winter gloves”
	•	“Try: 10-min morning yoga”
	2.	Анімація: коли слова наближаються до центру — вони розчиняються (opacity → 0) і ніби “всмоктуються” в ядро.
	3.	Після “всмоктування” вони мають заново спалахнути з різних позицій/кутів/дистанцій.
	4.	Збагатити варіації: стартова дистанція, кут, швидкість.

Acceptance criteria
	•	Тексти не дублюють thought cloud (“I will remember this…” не використовувати тут).
	•	При наближенні до ядра слова зникають до 0 opacity (не проходять крізь центр як “слайди”).
	•	Новий цикл стартує з інших позицій/кутів (не виглядає як повтор однієї траєкторії).
	•	На старті секції одразу видно “живу” картину (не так, що “поки доскролиш — там тільки потім стає цікаво”).

⸻

4) Use cases list (Travel / Gifts / Movies / Reading List…)

Change request

Зараз це виглядає як “крапка + заголовок + 2 рядки”. Треба оживити і зробити менш “списково”:
	•	додати рух обʼєктам (micro drift / float / subtle collisions)
	•	зробити маркери не однаковими синіми крапками:
	•	замінити на spark-markers (маленькі вогники)
	•	кожен use case має трохи свій характер (розмір/глоу/пульсація)
	•	зберегти текст (SEO), але надати “сценічність”

Acceptance criteria
	•	Маркери не однакові сині точки — це spark-вогники.
	•	Кожен пункт має мінімальний motion (drift/pulse), але читабельність не страждає.
	•	Не виглядає як “звичайний список” з темплейта.

⸻

5) Testimonials (критично): НЕ cards + spark ratings

Change request

Потрібно повністю прибрати “звичайні картки” і повернути floating testimonials, як у старій версії:
	•	layout: плаваючі відгуки в просторі (depth, slight parallax)
	•	додати:
	•	імʼя
	•	роль/контекст (1 рядок)
	•	рейтинг = 5 spark particles, НЕ ⭐ і НЕ статичні іконки
	•	spark-rating має бути “нерівний”: кожна іскорка трохи відрізняється + micro jitter/dribble

Acceptance criteria
	•	Відгуки не в card UI (жодних типових контейнерів).
	•	Відгуки “floating” (відчуття глибини) + не перекриваються.
	•	Rating: 5 spark-вогників, кожен з легким micro-motion (не синхронно).
	•	Додані name + role.
	•	На desktop немає накладань/перекриттів, на mobile — читабельно 1 за 1.
